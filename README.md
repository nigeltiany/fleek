# Fleek

A Flutter dating app with encrypted messaging. Older project — early code quality, kept around as an artifact of what I was learning at the time.

## What it does

- Tinder-style swipe-to-match dating UX, built with Flutter (iOS + Android).
- Per-recipient encrypted messaging — text, files, and audio.
- Profile setup, student-status verification, geolocation-based discovery, push notifications.

## How the encryption works

- Each user gets an asymmetric keypair generated by a Firebase Function (`createKeyPair`). The public key is shared; the private key is returned to the device and stored in `flutter_secure_storage` (OS keystore — Keychain on iOS, Keystore on Android).
- Chat content is stored in Firestore as a map keyed by recipient `userID`, with each entry encrypted under that recipient's public key using **ECIES** (Elliptic Curve Integrated Encryption Scheme — the `Gecies` calls in the code).
- File payloads (audio, images, attachments) are encrypted with **AES-GCM** via the `cryptography` Dart package, with a random secret key and nonce per file. The wrapped file-secret is itself transmitted via the ECIES channel.
- On read, the device fetches the recipient-keyed ciphertext, decrypts with the local private key, and renders.

### Honest limitations

This is **not** end-to-end encrypted in the strict sense:
- Keypairs are generated server-side by a Firebase Function rather than on-device, so the server theoretically observes the private key at issuance. A true E2E implementation would generate keys locally and never expose the private half.
- No forward secrecy — keys are long-lived per user, not per-session.
- No key verification UI (no fingerprint or QR comparison between users).

Treat the encryption layer as **server-issued asymmetric encryption with secure on-device key storage**, not as a Signal-grade E2E protocol.

## Stack

- Flutter / Dart (target SDK ~Dart 2.8)
- Firebase: Auth, Firestore, Cloud Functions, Storage, Messaging
- `cryptography` (AES-GCM), `flutter_secure_storage` (OS keystore), `geoflutterfire` (geo queries)
